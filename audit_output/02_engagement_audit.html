Drop-Off and Stuck Points

1. Psych Intake — The Biggest Risk HIGH IMPACT FIXED

Problem: The psych intake is a 10-15 message conversation with no progress indicator, no save-resume clarity, and no way to skip.

Resolution:

  • Progress indicator added: "Getting to know you (X of ~12)"
  • Continue Later button added to intake chat header
  • "Welcome back" message shown on resume
  • INTAKE_LOCKED now shows countdown timer with close button — no longer bricks the app

2. Baseline Lift Test — Tedious for Beginners MEDIUM IMPACT

Problem: 9 sequential lift tests requiring the user to physically load a barbell and perform max reps. No way to skip individual lifts. No "I don't know this exercise" option. No video/form guidance.

  • BASELINE_LIFTS includes barbell hip thrusts — an exercise many beginners don't know
  • The dot-progress indicator helps but there's no time estimate
  • Each lift is saved immediately — good for data safety, but the user doesn't know this

3. Morning Check-in — Blocks the Entire App MEDIUM IMPACT FIXED

Problem: The morning check-in overlay covers the entire screen with no dismiss option. The user MUST complete it before accessing their workout.

Resolution: Skip button added to the morning check-in overlay. Users can now bypass the check-in and go straight to their workout.

4. Coach Chat — No Conversation Starters LOW IMPACT FIXED

Problem: The chat overlay opens to an empty state or old messages. There's no prompt, no suggested questions, no quick-action buttons.

Resolution: Inline coach message now displayed on the main screen with quick-reply buttons ("How should I modify today?" / "I'm struggling" / "I crushed it"). Users no longer need to open a separate overlay to interact with the coach.

Latency Bottlenecks

1. Synchronous Claude API Calls CRITICAL MITIGATED

Where: Every coach message, every intake message, every photo analysis, every intake report generation.

Mitigation applied: Timeouts added to all API calls (coach 30s, psych chat 30s, psych report 60s, photo analysis 45s). Calls will no longer hang indefinitely. Calls are still synchronous — full async/streaming is a future improvement.

2. 8 Parallel Fetches on Load MEDIUM

Where: DOMContentLoaded handler (app.js:1162-1171)

Impact: Well-optimized via Promise.all(). However, the Garmin status check + subsequent refreshGarmin() happens sequentially after, adding 1-3 seconds to perceived load time.

3. Photo Upload + Analysis MEDIUM

Impact: Photo compression happens client-side (good), but then the base64 image is sent in a JSON POST body, followed by synchronous Claude Vision analysis. Total time: 5-15 seconds per photo.

Mitigation already in place: Photos show "AI Coach is analyzing..." with a spinner. This is good feedback.

4. Coach Context Assembly LOW

Impact: Queries 6 different database tables synchronously. For a mature user with months of data, the body weight query could return hundreds of rows. Not a problem yet, but will degrade.

Missing Feedback Loops

1. Weight and Completion Saves — No Confirmation

When a user logs a weight or toggles a completion checkbox, apiPost() fires and forgets. If the network is flaky (common on phone in a gym), data is silently lost.

Fix: Add a subtle toast or checkbox color flash on successful save. Show a red indicator on failure with retry.

2. Supplement Toggles — No Daily Summary

Users can toggle supplements throughout the day, but there's no end-of-day summary or reminder for missed supplements.

Fix: Add a "You missed: Fish Oil, Vitamin D3" nudge in the evening, or have the Coach mention it.

3. No Progress Milestones or Celebrations

The app tracks weight loss, lift progression, and completion rates, but never proactively celebrates milestones. No "Bench press PR!" or "Perfect week!"

Fix: Check for milestones in renderAll() and show a brief banner. Feed milestones into Coach context.

4. No "Streak" Visibility

The app tracks daily completion but there's no streak counter, no week-over-week comparison, no visual progress toward the 12-week goal.

5. Garmin Connection — No Nudge to Connect

The bar shows "Connect Garmin Watch" with no explanation of WHY connecting matters.

Top 3 Engagement Improvements

1. Add Coach-Initiated Morning Briefing HIGH IMPACT MEDIUM EFFORT DONE

What: Instead of a silent check-in form, open the morning with the Coach greeting the user based on yesterday's data and today's plan. "You slept 7.5 hours, body battery is 68. Today is bench day. How are you feeling?"

Why: The current flow is: fill out form → wait for coach → get generic response. Flipping it so the Coach speaks first makes the interaction feel like a real coaching session, not a medical intake form.

How: Pre-fetch Garmin data before showing the check-in overlay. Show the Coach's opening message first, then show sliders as the user's response to the Coach's question.

2. Fix the Intake Dead-End and Add Progress Indicator HIGH IMPACT LOW EFFORT DONE

What:

  • ✓ Add a message counter to the intake chat ("Getting to know you (X of ~12)")
  • ✓ Add a "Continue Later" button that saves and dismisses
  • ✓ Handle INTAKE_LOCKED with a proper UI: countdown, close button, return path
  • ✓ Show "Welcome back" on re-entry

3. Add Milestone Detection and Celebration MEDIUM IMPACT LOW EFFORT DONE

What: Detect and surface achievements: first week complete, weight PR on any lift, body weight milestones (every 2 lbs lost), streak achievements (7 days, 14 days), perfect adherence weeks.

Why: The app collects rich data but never reflects it back as accomplishments. The Coach's personality is "honest, direct, no hollow praise" — but SPECIFIC recognition of REAL achievements is part of the coaching philosophy ("Real breakthrough → recognize it specifically").

How: Add a checkMilestones() function to renderAll() that compares current data against stored thresholds. Show milestones as brief banners. Feed them into Coach context.
