1. Executive Summary

The coach (Erik) is architecturally sound and voice-consistent at its core. The Lombardi persona is well-defined in both the intake system prompt and the ongoing coaching prompt, with clear behavioral rules, escalation paths, and anti-cheerleading constraints. The scripted intake sequence (9 questions, 3-strike pushback, 7-day lockout) works well for compliant and moderately resistant users. However, the simulation reveals three systemic risks: (1) the coach has no proactive touchpoint between intake completion and morning check-in — the user finishes onboarding and hits a dead wall with no "what's next" from Erik, (2) the morning briefing prompt is a single sentence generator disconnected from the rich coach persona, creating a jarring voice break, and (3) post-workout feedback depends entirely on the user initiating chat — the coach never reaches out. The strongest signal is the intake system prompt's handling of answer acceptance (short answers, slang, humor all count as valid engagement) which prevents the most common AI coaching failure: treating casual users as hostile. The weakest signal is calibration — the coach has no mechanism to adjust intensity based on user archetype during intake, meaning a fragile Casey gets the same energy as a confident Alex.

Appendix: Methodology

What this report is: A predictive simulation based on analysis of the actual codebase. Every coach response was generated by reading the system prompts in psych_intake.py (lines 8-110) and coach.py (lines 115-194), understanding the behavioral rules, and predicting what Claude Opus would output given those prompts and the simulated user input.

What this report is not: A live test. No API calls were made. No real Claude responses were generated. The transcripts represent the most likely responses given the prompt design, based on deep familiarity with Claude's behavioral patterns under system prompts of this type.

Scoring criteria:

  • Voice fidelity (1-10): Does it sound like Lombardi/Goggins/Brooks or a wellness chatbot? 10 = indistinguishable from a real tough coach. 1 = generic AI assistant.
  • Specificity (1-10): Does the response reference actual user data, or could it be said to anyone? 10 = cites specific numbers, names, history. 1 = could be a template.
  • Presence (1-10): Does it feel like talking to a human who remembers you? 10 = eerily human, uses context seamlessly. 1 = clearly a bot following a script.
  • Resilience (1-10): Does the coach stay in character under pressure? 10 = unshakeable. 1 = breaks character at first pushback. N/A for non-resistant users.
  • Calibration (1-10): Is the intensity right for THIS user at THIS moment? 10 = perfectly matched. 1 = dangerously mismatched (crushing a fragile user or coddling a resistant one).

  • models.py — Database models for all user data
